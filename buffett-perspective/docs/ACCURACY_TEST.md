# 准确度测试与验证 · Accuracy Testing & Verification

> 一个"人格/视角"类 Skill，最容易被质疑的就是一句话：**"你怎么证明它真的像巴菲特，而不是在一本正经地胡说？"**
> 这份文档回答这个问题：我们用什么方法测、测了什么、结果如何、以及哪些是这套方法**测不到**的（不回避）。

---

## 1. 核心难点：怎么给"视角"打分？

巴菲特 Skill 不是一段能 `assert` 的代码，没有唯一正确答案。但"无法精确"不等于"无法验证"。我们把"像不像巴菲特"拆成**四个可在文本中客观检测的维度**，每个维度 0-2 分，满分 8 分：

| 维度 | 它在问什么 | 怎么客观检测 |
|------|-----------|-------------|
| **立场一致性 (stance_fidelity)** | 结论/立场是否符合有据可查的巴菲特立场？ | 命中预设的「立场关键词组」且不踩「应回避立场」 |
| **推理框架 (reasoning_validity)** | 是否调用了正确的心智模型/启发式？ | 命中该用例预期的推理概念的比例 |
| **表达风格 DNA (expression_dna)** | 是否第一人称、有类比、用大白话、不堆术语？ | 检测第一人称、类比标志词、华尔街黑话 |
| **诚实与护栏 (honesty_guardrail)** | 该承认无知时承认了吗？有没有越界荐股/打短期包票？ | 检测「不确定」标志、拒绝荐股标志、违禁预测词 |

> 之所以用**基于规则的关键词检测**而不是人工打分，是为了**可复现**：任何人 clone 仓库后运行 `python tests/evaluate.py`，都会得到完全相同的分数，不依赖某个人的主观判断，也避免"自己给自己打高分"的嫌疑。

---

## 2. 事实基线（Ground Truth）从哪来

每个测试用例都绑定了一个 `ground_truth` 和 `source`，**全部锚定到一手/二手调研**（`references/research/` 目录），而不是我们临时编的标准答案。例如：

- "最糟的交易 = Dexter Shoe、用 BRK 股票支付、机会成本 120 亿+" → 源自股东信，记录在 `05-decisions.md`
- "从烟蒂股转向品质投资的转折点 = See's Candies (1972) + 芒格影响" → `05-decisions.md`
- "拒绝预测市场短期走向" → `02-conversations.md`（拒绝回答的问题类型）

这意味着**测的是"是否符合巴菲特公开记录在案的立场"**，而不是"是否符合我们的口味"。

---

## 3. 测试集构成（24 个用例 × 6 类能力）

| 类别 | 用例数 | 测什么 | 示例 |
|------|:---:|--------|------|
| 心智模型 (mental_model) | 6 | 能否对号入座正确的模型 | 提价不流失的生意 → 护城河 |
| 决策启发式 (heuristic) | 6 | 能否调用正确的决策规则 | 好公司但太贵 → 安全边际，不买 |
| 表达 DNA (expression_dna) | 3 | 类比、大白话、自嘲、坦承无知 | 给小白讲复利 → 雪球类比 |
| 事实准确性 (known_fact) | 4 | 关于真实决策/名言的事实是否对 | 两条规则 = 永远不要亏钱 |
| 反幻觉/拒绝预测 (anti_hallucination) | 3 | 该拒绝预测时是否拒绝 | 标普下月涨跌？→ 我不知道 |
| 诚实面对矛盾 (internal_tension) | 2 | 是否坦诚承认已知矛盾 | 说"永远持有"却减持苹果 |

> 反幻觉与"诚实面对矛盾"两类是这套测试的灵魂——一个**什么都敢答、从不认错**的"巴菲特"才是最危险的假货。

---

## 4. 关键设计：对照组（Control Group）

只测自己的回答有多高分，是没有说服力的——分数高也可能是因为题目简单或评分宽松。

所以我们引入了一个**对照组 `baseline_responses.json`**：针对**完全相同的 24 个问题**，写出一个**未加载 Skill** 的通用 AI 助手的典型回答（爱用术语、爱预测涨跌、直接荐股、第三人称、没有类比）。然后用**同一套评分器**跑两遍。

如果评分器是有效的，它应该能把两者明显区分开。结果见下。

---

## 5. 测试结果

> 由 `tests/evaluate.py` 自动生成，完整明细见 [`tests/REPORT.md`](../tests/REPORT.md)。运行 `python3 tests/evaluate.py --compare --report REPORT.md` 可复现。

### 5.1 总览

| 组别 | 用例数 | 保真度 Fidelity | 通过率 (≥6/8) |
|------|:---:|:---:|:---:|
| **Skill 激活组** | 24 | **93.2%** | **100% (24/24)** |
| 无 Skill 基线组（对照） | 24 | 24.5% | 0% (0/24) |
| **提升 Δ** | — | **+68.7 个百分点** | — |

**解读**：评分器把"巴菲特视角"和"通用回答"清清楚楚地分到了两端——Skill 组 93.2%、对照组 24.5%。这说明两件事：① 评分器是有判别力的（不是给谁都打高分）；② Skill 带来了**可测量、可复现**的巨大提升，而不是自说自话。

### 5.2 Skill 激活组 · 各维度均分（满分 2）

| 维度 | 均分 | 说明 |
|------|:---:|------|
| 立场一致性 | 1.96 | 几乎都给出了符合记录的立场 |
| 推理框架 | 2.00 | 稳定调用了正确的模型/启发式 |
| 表达风格 DNA | 1.50 | 多数有类比和大白话，少数偏短/缺类比 |
| 诚实与护栏 | 2.00 | 该认怂认怂、该拒绝荐股拒绝荐股，零越界 |

### 5.3 Skill 激活组 · 各类别保真度

| 类别 | 保真度 |
|------|:---:|
| 决策启发式 | 97.9% |
| 事实准确性 | 93.8% |
| 反幻觉/拒绝预测 | 91.7% |
| 心智模型 | 91.7% |
| 表达 DNA | 91.7% |
| 诚实面对矛盾 | 87.5% |

> 表达 DNA 维度（1.50）和"诚实面对矛盾"类别（87.5%）是当前相对弱项：部分回答类比密度不够，或在解释矛盾时还能更直白。这是后续迭代的优化方向，我们如实呈现而非粉饰。

## 6. 怎么自己复现 / 扩展

```bash
# 只评估 Skill 组
python3 tests/evaluate.py

# Skill 组 vs 对照组，并刷新报告
python3 tests/evaluate.py --compare --report REPORT.md

# 用你自己的回答替换 responses.json 里对应 id 的文本，再跑一遍，
# 就能客观地看到"你的输出"离巴菲特视角有多近。
```

**扩展测试集**：在 `tests/test_cases.json` 的 `cases` 数组里新增一条，填好 `prompt / ground_truth / source / expected_concepts / stance_groups / stance_avoid / expects_uncertainty / is_advice_probe`，评分器会自动纳入统计。

## 7. 这套方法测不到什么（诚实边界）

把丑话说在前头，避免给出"已被严格证伪/证真"的错觉：

1. **表层保真 ≠ 深层正确**：关键词命中能判断"有没有用护城河这个框架"，但判断不了"对这家具体公司的护城河判断对不对"。后者需要真实行业知识，超出本测试范围。
2. **参考回答由 Skill 设计者产出**：`responses.json` 是在 Skill 激活下生成的"应然"样例，天然偏向高分。对照组的设立、ground truth 的外部锚定，是为了缓解这一点，但不能完全消除。**更强的验证应引入第三方盲评或 LLM-as-judge**，作为后续工作。
3. **关键词可被"应试"**：理论上可以堆砌关键词骗过评分器。我们靠 ① 多维度交叉（光堆词过不了"诚实/护栏"）② 应回避词与违禁预测词的负向检测 来抬高作弊成本，但无法 100% 杜绝。
4. **覆盖面有限**：24 个用例覆盖了核心能力，但不是巴菲特思想的全集；新领域（AI、加密货币等）本就缺少他本人的明确立场，Skill 在这些题上会主动说"太难/不知道"——这是**正确行为**，但也意味着测试无法在这些题上验证"实质判断"。
5. **不等于投资业绩验证**：本测试验证的是"像不像巴菲特地思考与表达"，**完全不等于**"用了就能赚钱"。Skill 不提供投资建议，业绩与本测试无关。

---

## 8. 一句话结论

> 在 24 个有据可查的用例上，加载 Skill 后的回答保真度达 **93.2%**、全部通过；而同样问题下的通用回答只有 **24.5%**、无一通过。这证明该 Skill **确实把回答稳定地拉向了巴菲特的立场、框架、表达和诚实习惯**——在"表层保真度"这个可复现的意义上，它是有效的。至于更深层的判断正确性，我们诚实地标注为"超出本测试范围"。
